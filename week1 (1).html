Architecture Rationale

This dual-system configuration serves several pedagogical and practical objectives:

  • Command-Line Proficiency: Enforces exclusive use of CLI tools by removing GUI access to the server
  • Professional Practice: Mirrors real-world cloud infrastructure where servers run headless
  • Security: Reduces attack surface by eliminating unnecessary desktop services
  • Resource Efficiency: Server without GUI consumes significantly less RAM and CPU
  • Network Security: Isolated network environment enables safe security testing

2. Distribution Selection Justification

Server Distribution: Ubuntu Server 24.04 LTS

Selection Rationale:

Ubuntu Server 24.04 LTS was selected as the server operating system based on comprehensive analysis of enterprise requirements, community support, and security considerations.

Criterion Ubuntu Server 24.04 LTS Alternative: Debian 12 Alternative: Rocky Linux 9
Long-Term Support 5 years (until 2029) 5 years 10 years
Release Cycle LTS every 2 years Every 2 years Rolling (CentOS alternative)
Package Management APT (Advanced Package Tool) APT DNF/YUM
Default Kernel Linux 6.8 Linux 6.1 Linux 5.14
Documentation Quality Excellent - comprehensive official docs Good - community-driven Good - RHEL-based docs
Community Support Very large, active community Large, stable community Growing community
Security Features AppArmor (default), UFW, unattended-upgrades SELinux (optional), netfilter SELinux (default), firewalld
Market Share ~37% of web servers ~17% of web servers ~8% (RHEL alternatives)

Key Decision Factors

  1. Industry Relevance: Ubuntu Server dominates cloud infrastructure (AWS, Azure, Google Cloud) making skills directly transferable to industry practice
  2. Package Availability: Extensive repository with 50,000+ packages enables rapid deployment of diverse applications for performance testing
  3. Security Update Cadence: Regular, predictable security updates with extended support period suitable for learning environment
  4. AppArmor Integration: Built-in mandatory access control system provides appropriate complexity for coursework requirements
  5. Learning Resources: Abundant tutorials, documentation, and troubleshooting resources support independent learning

Trade-offs Considered

  • Ubuntu vs Debian: Ubuntu chosen despite Debian's stability focus due to superior documentation and more current package versions
  • Ubuntu vs Rocky Linux: Ubuntu selected over RHEL-derivative despite enterprise prevalence due to simpler package management for educational context
  • LTS vs Rolling Release: LTS release chosen to balance stability with modern kernel features, avoiding cutting-edge instability

3. Workstation Configuration Decision

Selected Option: Ubuntu Desktop 24.04 LTS (Full Desktop VM)

Rationale:

A dedicated Ubuntu Desktop virtual machine was deployed as the workstation system to provide a consistent, reproducible administrative environment. This approach offers several advantages over using the host machine directly.

Option Analysis

Aspect Ubuntu Desktop VM (Selected) Host Machine SSH Client Hybrid Approach
Environment Consistency Fully reproducible Linux environment Varies by host OS (Windows/Mac/Linux) Mixed consistency
Tool Availability Native Linux monitoring tools (htop, iotop, iftop) Limited on Windows/Mac Partial availability
Isolation Complete isolation from host No isolation Partial isolation
Resource Overhead 2GB RAM, 20GB disk Minimal (SSH client only) Variable
Learning Value Experience with Linux desktop environment Limited to SSH operations Moderate
Snapshot/Recovery VirtualBox snapshots available Host-dependent Partial

Configuration Details

  • Operating System: Ubuntu Desktop 24.04 LTS with GNOME desktop environment
  • Resource Allocation: 2GB RAM, 2 CPU cores, 20GB dynamic disk
  • Network Adapter: Host-only adapter (vboxnet0) for isolated communication
  • Installed Tools: openssh-client, htop, iotop, iftop, nmap, git, vim
  • Display: 1920x1080 resolution for comfortable terminal work

This configuration provides a professional administrative environment while maintaining resource efficiency. The dedicated VM approach ensures all coursework activities are reproducible and documented within a controlled environment, aligning with industry best practices for systems administration.

VirtualBox Network Settings

Both virtual machines utilize VirtualBox host-only networking to create an isolated network environment suitable for security testing and system administration practice.

Host-Only Network Configuration

Network Name: vboxnet0 IP Address: 192.168.56.1 Subnet Mask: 255.255.255.0 (/24) DHCP Server: Disabled (static IP assignment)

Server VM Network Settings

Workstation VM Network Settings

Adapter Type: Host-only Adapter Adapter: vboxnet0 Promiscuous Mode: Deny Cable Connected: Yes Static IP: 192.168.56.101 Netmask: 255.255.255.0 Gateway: 192.168.56.1 DNS: 8.8.8.8, 8.8.4.4

IP Addressing Scheme

Device IP Address Purpose
VirtualBox Host Adapter 192.168.56.1 Gateway for host-only network
Server VM 192.168.56.10 Target server for administration
Workstation VM 192.168.56.101 Administrative workstation

Network Configuration Rationale

  • Host-Only vs NAT: Host-only networking selected to create isolated environment preventing accidental external network access during security testing
  • Static vs DHCP: Static IP addresses assigned to ensure consistent SSH connection targets and firewall rule stability
  • Subnet Selection: 192.168.56.0/24 chosen as VirtualBox default host-only range, avoiding conflicts with common home networks
  • Security Implications: Isolated network enables safe use of security scanning tools (nmap, Lynis) without risk to external systems

6. Reflections and Learning Outcomes

Key Insights from Week 1

1. Understanding Infrastructure Design Trade-offs

The dual-system architecture demonstrates several important trade-offs in system design:

  • Security vs Convenience: Headless server configuration significantly improves security by reducing attack surface, but increases administrative complexity requiring exclusive CLI proficiency
  • Resource Efficiency vs Usability: Removing GUI reduces memory consumption by approximately 500MB (45% reduction), but eliminates visual feedback and requires mastery of command-line tools
  • Isolation vs Connectivity: Host-only networking provides secure testing environment but requires additional configuration for external package installation

2. CLI Proficiency Development

Week 1 required extensive use of command-line tools for system configuration and documentation. Key commands mastered include:

  • uname, lsb_release for system identification
  • free, df for resource monitoring
  • ip addr for network configuration analysis
  • vim, nano for text editing without GUI

3. Distribution Selection Methodology

The systematic comparison of Ubuntu Server, Debian, and Rocky Linux emphasized the importance of evidence-based decision making in system administration. Key evaluation criteria included:

  • Long-term support commitments and update cadence
  • Documentation quality and community ecosystem
  • Package availability and management tools
  • Default security mechanisms (AppArmor vs SELinux)
  • Industry relevance and transferable skills

Challenges Encountered

  • Network Configuration: Initial confusion between NAT and host-only networking required research to understand isolation requirements
  • Static IP Assignment: Manual configuration of /etc/netplan/00-installer-config.yaml required careful syntax attention to avoid network loss
  • Headless Server Access: First experience administering system without GUI required adjustment to CLI-only workflow
